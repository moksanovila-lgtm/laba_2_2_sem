Почему в моём предыдущем ответе был include/Sequence.tpp
Я предположил стандартную для C++ структуру:

include/ — для всего, что подключается через #include (.h, .hpp, .tpp, .inl)

src/ — для .cpp файлов, которые компилируются отдельно

Так как .tpp подключается через #include, он должен лежать в include/.

Почему .tpp в include/, а не в src/
Файл	Подключается через #include?	Должен лежать в
DynamicArray.h	Да (пользователем)	include/
DynamicArray.tpp	Да (из DynamicArray.h)	include/
main.cpp	Нет (компилируется отдельно)	src/ или корень
Ключевое правило: Любой файл, который попадает в другой через #include, должен лежать в директории с заголовками.

Как выглядит правильное включение
cpp
// DynamicArray.h
#ifndef DYNAMIC_ARRAY_H
#define DYNAMIC_ARRAY_H

template <typename T>
class DynamicArray {
    // объявления
};

#include "DynamicArray.tpp"  // ? подключает файл из той же директории

#endif
Если DynamicArray.tpp лежит в src/, этот #include не сработает (или придётся писать #include "../src/DynamicArray.tpp", что плохо).
